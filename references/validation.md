# Валидация хай-концепта — 11 критериев

> Концепт считается **сильным**, если проходит **≥9 из 11 критериев** и содержит **0 слов из anti-slop blacklist** (см. anti-patterns.md). Если <7/11 — отбраковывается, нужен regenerate.

## 11 критериев — основной rubric

| # | Критерий | Как проверить |
|---|----------|---------------|
| 1 | **Объясняется за 10 секунд** | Прочитать вслух. Умещается в 10 секунд при обычном темпе речи. |
| 2 | **Помещается в одно предложение** | Одна точка в конце. Нет двух самостоятельных предложений склеенных «и» / «—» / «,». |
| 3 | **Вызывает эмоцию** | Цепляет. Не «гигиенический», не нейтральный. Можно почувствовать что это про конкретного человека. |
| 4 | **Провоцирует вопрос «как это?»** | Есть крючок, после которого хочется спросить подробнее. Не сразу всё объясняет. |
| 5 | **Обесценивает конкурентов без прямой атаки** | Делает стандартное предложение конкурентов «не таким». Без называния имён. |
| 6 | **Заставляет конкурентов оправдываться** | Если конкурент услышал — ему пришлось бы объяснять, «почему у нас не так». |
| 7 | **Меняет критерий сравнения** | Создаёт новую ось сравнения (не цена / скорость / качество, а другая). |
| 8 | **Естественно генерирует UGC/пересказ** | Хочется поделиться с другом / коллегой. Запоминается. |
| 9 | **Работает как позвоночник** | Из концепта выводится Hero-копирайт, About-секция, messaging для соцсетей. Не ограничивается одной фразой. |
| 10 | **Есть конкретная боль** | В концепте есть слово / образ из мира боли клиента. |
| 11 | **Есть понятный результат** | В концепте есть слово / образ из мира желаемого состояния. |

### Scoring

- **≥9/11 + 0 slop-words** → принимаем
- **7-8/11 + 0 slop-words** → можем показать, но желательно refine
- **<7/11 или ≥1 slop-word** → отбрасываем, регенерим

## Дополнительные проверки

### Brand-swap test (критичен для product brands)

Убери из фразы название продукта и нишу. Если она всё ещё звучит **осмысленно для конкурента** — это slop.

**Плохо:**
```
«Надёжное решение для вашего бизнеса»
→ «Надёжное X для Y» → подставимо под любой → SLOP
```

**Хорошо:**
```
«CRM для автосервисов 5-20 мест без абонплаты»
→ «X для Y без Z» → теряет суть если изменить → НЕ slop
```

Для personal brand brand-swap менее строг (имя владельца часто и есть имя бренда), но если фраза слишком общая («Я помогаю людям менять жизнь к лучшему») — это slop того же типа.

### Grunt Test (Donald Miller)

Понял бы 3-классник? Если в концепте есть профессиональный жаргон, аббревиатуры, академическая лексика — Grunt Test провален.

- ❌ «Cognitive behavioral framework для трансформации executive mindset»
- ✅ «Помогаю руководителям перестать тащить всё на себе»

### SUCCESs (Chip & Dan Heath)

Концепт лучше «прилипает», если выполняет 6 принципов:

- **S**imple — одна центральная идея
- **U**nexpected — есть элемент неожиданности / нарушения паттерна
- **C**oncrete — конкретные образы / факты, не абстракции
- **C**redible — звучит правдоподобно, есть подкрепление
- **E**motional — про чувства, не статистику
- **S**tories — встроена история / сценарий

Не все 6 обязательны. 3-4 из 6 = хорошо.

### Ownability test (Tagline Principles)

Уберите название бренда из концепта. Можно ли его подставить под конкурента? Если **да** — концепт слабый (не «owned» вашим брендом).

«**Nike**: Just Do It» → «**Adidas**: Just Do It» → звучит **не так** для Adidas (Adidas про коллективное превосходство, не индивидуальное преодоление) → ownable.

«**Любой бренд**: Премиальное качество и индивидуальный подход» → подставимо под любой → НЕ ownable → slop.

### Memorability test (Heath)

Пересказывается ли концепт **через сутки**? Если после прочтения вечером пользователь утром не может вспомнить — концепт не прилипает.

Признаки memorable:
- Короткий (3-7 слов часто)
- Конкретные образы (не абстракции)
- Контраст / неожиданность
- Звуковой ритм (рифма / аллитерация — бонус)

## Что делать с критикой судьи

LLM-судья ([`prompts/judge.md`](../prompts/judge.md)) возвращает:

```json
{
  "evaluations": [
    {
      "variant_id": "pain_relief",
      "criteria": {
        "criterion_1_ten_seconds": true,
        "criterion_2_one_sentence": true,
        ...
      },
      "score": 7,
      "slop_words": [],
      "critique": "Концепт слабоват в эмоции (3) и не создаёт новую ось сравнения (7). В следующей итерации усилить контраст с тем что предлагает рынок сейчас."
    },
    ...
  ]
}
```

**Если все варианты <7/11:**
- Прочитать critique
- Сделать Self-Refine: вернуть варианты в генератор с критикой как feedback
- Максимум 2 refine-итерации
- После 2-й — safe degradation: показать самый сильный из имеющихся, даже если не идеален

**Если 1+ вариант ≥9/11 + 0 slop:**
- Показать пользователю
- Слабые варианты можно тоже показать как «для сравнения», но пометить как «слабее по критериям»

## Чек-лист перед финализацией

Перед тем как сказать «концепт готов»:

- [ ] ≥9/11 критериев пройдены
- [ ] 0 слов из anti-slop blacklist
- [ ] Brand-swap test пройден (для product)
- [ ] Grunt Test пройден (нет жаргона)
- [ ] Концепт ownable (нельзя подставить конкурента)
- [ ] Из концепта понятно как делать Hero / About / messaging
- [ ] Test_phrase — реально то что пользователь сказал бы другу

Если все галочки — концепт готов к downstream-задачам.
